# Preferences

## Communication Style

- Be direct and concise — skip the fluff
- Use bullet points over long paragraphs
- Code examples over lengthy explanations
- Don't over-explain things I already know

## Work Style

- I work best in focused 2-hour blocks (no Slack, no distractions)
- Morning person — most productive before noon
- Prefer async communication; I check messages 2x per day
- I like written specs before jumping into code

## AI Interaction Preferences

- Don't start responses with filler phrases ("Great question!", "Certainly!")
- Give me the answer first, explain after if needed
- Show full code blocks, not partial snippets with `...`
- If something is ambiguous, ask one clarifying question rather than assuming

## Tech Preferences

- TypeScript over JavaScript always
- Tailwind over CSS modules
- Vercel for deployment
- Supabase over Firebase
- Prefer simple solutions over clever ones

## Tools I Use Daily

- VS Code + Cursor
- Linear for task tracking
- Figma for design
- Arc browser
